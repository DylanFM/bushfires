Fire incidents API

An API to current and previous RFS major incidents.

¶

Resource¶

GET/incidents

Retrieve a collection of incidents. Currently, in lieu of pagination, a time range is required for all requests.

Parameters
HideShow
timeStart
timestamp (required) Example: 2014-01-01T00:00:00+00:00

Start of time range in UTC (ISO 8601)

timeEnd
timestamp (required) Example: 2014-01-31T23:59:59+00:00

End of time range in UTC (ISO 8601)

Response  200
HideShow

JSON conforming to the GeoJSON spec. Response is a FeatureCollection containing 0 or more features. Each Feature represents an incident. Properties included in the feature object are taken from the incident’s most recent report.

This endpoint has the ablity to return very large responses. In the interest of performance, the features within the response contain minimal information. They will not include geographic polygons and all of an incident’s properties. They will include coordinates for the incident and an ID for retrieving all of the incident’s details.

The incidents included in the response are those that were current within the requested time range. E.g if you request one day’s worth, it may include incidents that were current for an hour during that day, some that had been current for days beforehand but finished on that day, and some that were current for many days before and after.

Resource¶

GET/incidents/{id}/reports

Retrieve all reports for a specified incident.

Parameters
HideShow
id
string (required) Example: df34a81e-4fe5-4c09-8403-466526ea503c

Id of an incident.

Response  200
HideShow

Reports are returned as a GeoJSON FeatureCollection whose features are ordered by pubdate in ascending order. Each item within the FeatureCollection is a Feature representing the report. A report is basically a snapshot of the incident at a given time. A report should never be identical to the one preceeding or following it.
